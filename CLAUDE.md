# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`shgo-rs-native` (GitHub: `jpswensen/shgo-rs-native`) is a native Rust port of SciPy's SHGO
(Simplicial Homology Global Optimization) algorithm. The reference implementation is
`scipy/optimize/_shgo.py`; the guiding principle is behavioral fidelity to SciPy, with
documented extensions (rayon parallelism). Do not confuse this repo with the sister project
`jpswensen/shgo-rs`, which is a pyo3 wrapper that calls SciPy's Python SHGO from Rust.

Naming trap: the *package* is `shgo-rs` but the *library target* is `shgo`
(`[lib] name = "shgo"`), so imports are `use shgo::{Shgo, ShgoOptions};`. Doc-comment
examples that write `use shgo_rs::…` fail to compile as doctests.

## Commands

```bash
cargo build --release              # fat LTO + codegen-units=1: slow link, expected
cargo test                         # 103 unit + 11 cross-validation + 21 doctests, all green
cargo test --release <test_name>   # run a single test by substring
cargo test --test cross_validation # Rust vs Python fixture comparison only
cargo bench                        # criterion benchmarks (benches/benchmarks.rs)
cargo run --release --example parallel_benchmark   # see examples/ for others
cargo run --release --example fix_benchmark        # perf+accuracy regression harness (JSON lines)
```

Note: in parallel test runs the lib binary's final `test result:` summary line is
sometimes swallowed from captured output (flush artifact); rerun with
`-- --test-threads=1` if you need the tally — do not mistake the missing line for a
failure (cargo's exit code is authoritative).

Cross-validation fixtures (`tests/fixtures/*.json`) are generated by
`python tests/generate_fixtures.py`, which requires the *Python* `shgo` package
(it path-hacks `../../shgo` expecting a checkout of stefan-endres/shgo, or falls back to an
installed package). Neither is currently available on this machine, so regeneration fails;
the checked-in fixtures are the source of truth until that is set up. End-to-end
fixtures (`tests/fixtures/end_to_end_tests.json`) are regenerated by
`tests/generate_e2e_fixtures.py`, which only needs `pip install scipy numpy` in a venv.

C/C++ examples: build the static lib first (`cargo build --release`), then use the compile
commands in the header comments of `examples_cpp/example.c` / `example.cpp`, e.g.
`gcc -O2 -I../include example.c ../target/release/libshgo.a -lpthread -ldl -lm -o example_c`.

## Architecture

Control flow: `Shgo::minimize()` (src/shgo.rs) validates bounds, optionally installs a rayon
pool sized by `options.workers` (None = all cores, `Some(1)` = serial/deterministic), then
dispatches on `SamplingMethod`:

- **Simplicial** (default): builds a `Complex` (src/complex.rs) via cyclic-product
  triangulation of the hyperrectangle (2^dim corners + centroid) and grows it with
  SciPy-parity semantics: every iteration adds ~n = 2^dim+1 new vertices
  (`Complex::refine(Some(n))` — bounded initial triangulation on iteration 1, then
  region-by-region C3 refinement (`refine_local_space_budgeted`) with a vertex budget;
  an interrupted region resumes via idempotent re-walk, mirroring SciPy's generator
  suspension). Growth parity vs scipy 1.18 is pinned by
  `test_simplicial_default_samples_initial_complex_only` and
  `test_simplicial_linear_growth_per_iteration` — keep those green when touching
  refinement. Beware: SciPy's *reported* `res.nfev` under-counts in multi-iteration
  simplicial runs (stale `fn`); compare against actual evaluation counts, not SciPy's
  report. Local minimizations use locally-convex bounds tightened by neighbor
  positions (`construct_lcb_simplicial`, exact port of SciPy's).
- **Sobol**: draws Joe-Kuo Sobol points (src/sobol.rs, via `sobol-qmc`, bit-exact vs
  `scipy.stats.qmc.Sobol` unscrambled — verified by fixtures), rounds n up to a power of 2,
  then builds vertex connectivity per `options.connectivity_method`
  (`ConnectivityMethod` enum): `Delaunay` (default, qhull with joggle retry on
  degeneracy; replicates SciPy's `vf_to_vv` *including its quirk* of only connecting the
  first two indices of each dim-combination), `KNearestNeighbors` (brute-force
  O(n²·d), bidirectional), `HNSW` (`hnsw_rs`, approximate), or `ScaNN` (`vecstore`,
  f32-quantized, falls back to KNN on failure). k defaults to `2·dim + 1`, overridable
  via `knn_neighbors`. All four connectivity methods build the graph over the **full
  cumulative point cloud** each iteration (matching SciPy's re-triangulation
  semantics; the KNN build is rayon-parallel; ScaNN query failures fall back to
  brute-force k-NN).
  Local minimization uses global bounds (matches SciPy's
  `construct_lcb_delaunay`). Each iteration continues the Sobol sequence
  (`skip = sobol_skip + total_points`) and re-inserts previously found minima as points.

Shared infrastructure:

- `VertexCache` (src/vertex.rs): coordinate-keyed `IndexMap` of `Arc<Vertex>` with
  double-checked locking. Vertex `index` equals insertion position (assigned under the
  write lock, vertices never removed) — `get_by_index` and the hot paths rely on this
  invariant for O(1) positional lookup (`IndexMap::get_index`, guarded by a
  `debug_assert` in `get_or_create`); any vertex-pruning change breaks it. New vertices
  go into pending pools; `process_pools()` batch-evaluates constraints then the
  objective in parallel (rayon). A vertex is a minimizer iff its f is strictly below all
  neighbors'; results are cached per-vertex and invalidated on connect/disconnect.
- `Coordinates` (src/coordinates.rs): hashable f64 vector using bit-representation equality
  (so `0.0 != -0.0`, NaN == NaN), pre-computed hash. Used as the dedup key everywhere.
- `LMapCache` (src/shgo.rs): thread-safe memo of local-minimization results keyed by
  *starting point*, which is what makes the parallel `par_iter` pool deterministic.
- `local_opt.rs`: NLopt wrapper. NLopt handles are `!Send`, so a fresh optimizer is built
  per call. The algorithm is selected via `local_options.algorithm` (BOBYQA default) —
  the old separate `ShgoOptions.local_optimizer` field was removed in commit 70ec80d;
  when constraints exist and the chosen algorithm can't handle them, SHGO auto-upgrades
  to COBYLA with a warning. SHGO's `g(x) >= 0` convention is negated to NLopt's
  `fc(x) <= 0`.
- `ffi.rs`: C API (`shgo_create`/`shgo_set_options`/`shgo_minimize`/…). The headers in
  `include/shgo.h`/`.hpp` are **manually maintained** — build.rs does not run cbindgen
  (cbindgen.toml exists only for manual regeneration). Any change to `ffi.rs` must be
  mirrored in both headers. Constraints registered via `shgo_add_constraint` are passed
  through to `Shgo::with_constraints` (regression test: `test_c_api_constraint_respected`).

Fidelity notes that matter when editing: stopping criteria mirror SciPy — if any of
maxiter/maxfev/maxev/maxtime/f_min is set, `iters` is disabled (`effective_iters`); the
f_min/f_tol formula matches SciPy's `finite_precision`; `result.nfev` = sampling
evaluations + `nlfev` (all local attempts, matching SciPy's
`res.nfev = fn + nlfev` — while `maxfev` is still checked against sampling evals only).
Local results enter `xl` whenever their `fun` is finite — converged or budget-capped —
matching SciPy's `LMC.add_res`, which appends every local result unconditionally; the
minimizer pool is sorted by f before `maxiter_local` truncation (`sort_min_pool` parity).

## Downstream consumers

The algotrader at `~/src/tcdts/trader` depends on this crate via
`shgo-rs-native = { git = "https://github.com/jpswensen/shgo-rs-native", package = "shgo-rs" }`
(so its crate ident there is `shgo_rs_native`), with Cargo.lock pinning a specific
commit. API changes here (e.g., the `local_optimizer` field removal) break that consumer
on its next `cargo update`. A separate, **divergent** checkout of this repo exists at
`~/src/tcdts/shgo-rs-native` carrying two unpushed experimental commits (a VertexCache
pruning perf experiment and its revert) — it is *not* used by the trader build (no path
patch) and lacks the connectivity work.
